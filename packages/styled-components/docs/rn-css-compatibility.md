# React Native CSS Features Compatibility

This is a living index of modern CSS capabilities and their current state on the React Native path. Features are grouped into what works today, what we polyfill, and what still needs upstream React Native support to become usable.

The goal is transparency: if you rely on something in this document, you can see whether it runs today, how we handle it under the hood, and what would need to change upstream to move a "not supported" row into the "polyfilled" or "native" column.

If a feature you need isn't listed, or the workaround section needs updating, open a comment or pull request to extend this file.

## Status snapshot

As of React Native 0.85:

- Many modern CSS features work natively, including CSS string transforms, box-shadow with spread/inset, filter chains, gradients, slash-alpha colors, `hwb()`, `mix-blend-mode`, `isolation`, `cursor`, `display: contents`, logical margin/padding/inset longhands, and several logical border longhands.
- Many features that React Native does not parse natively are polyfilled by styled-components: either converted before styles reach React Native or resolved against the live device environment.
- Other features still require upstream React Native work to support meaningfully. This document is a contract with the community: when one of those ships, we wire it up and move the row out of the unsupported section.

## Works natively in React Native 0.85

- `aspect-ratio`, `backface-visibility`, `border-radius`, `box-sizing`, `display: contents`, `object-fit`, `opacity`, `outline-offset`, `overflow`.
- `background-image: linear-gradient()` / `radial-gradient()` + `background-size` / `background-position` / `background-repeat`.
- `box-shadow` with offset, blur, spread, inset, and multi-shadow lists.
- Colors: hex, `rgb()` / `rgba()`, `rgb(r g b / a)` slash-alpha, `hsl()` / `hsla()`, `hwb()`, named colors, `transparent`.
- `cursor`, `direction`, `isolation`, `mix-blend-mode`, `pointer-events`, `user-select`.
- `filter: blur() brightness() contrast() drop-shadow() grayscale() hue-rotate() invert() opacity() saturate() sepia()` chains. On iOS, RN 0.85 requires the SwiftUI filter backend (`ReactNativeReleaseLevel` experimental) for `blur`, `saturate`, `hue-rotate`, `grayscale`, `contrast`, and `drop-shadow`; `brightness` and `opacity` work without that flag.
- Flex: `flex-grow` / `flex-shrink` / `flex-basis`, `align-items`, `justify-content`, `align-content`.
- Gap: `gap` / `row-gap` / `column-gap`.
- Logical margin / padding / inset longhands in the inline and block axes, plus their `-top` / `-bottom` / `-left` / `-right` counterparts. React Native also supports logical border colors/radii and start/end widths; styled-components fills the remaining `border-inline-*` / `border-block-*` CSS syntax below.
- Position: `absolute` / `relative`, plus `static` on the new renderer.
- `transform: translateX(10px) rotate(45deg) scale(1.2) matrix(<9 or 16 numbers>)` as a CSS string or transform array. React Native 0.85 accepts `matrix` arrays of 9 or 16 numbers, but the CSS string parser does not support the `matrix3d()` function name, and non-zero translate lengths need units. On Android, `skewX()` / `skewY()` parse but React Native drops the rendered skew; development builds warn once.

## Polyfilled by styled-components

These work in your style templates even though React Native doesn't parse them natively. styled-components either rewrites the value before handing it to React Native, or re-evaluates it on each render against the live environment.

When `styled-components/native` is bundled for `react-native-web`, the browser handles CSS surfaces it already supports where react-native-web can pass them through safely. That means `light-dark()`, viewport units including `dvh` / `svh` / `lvh` / `vi` / `vb`, and mixed-unit `calc()` / `min()` / `max()` / `clamp()` stay as CSS instead of being approximated through the iOS / Android polyfills. Static modern color functions are converted to sRGB hex before they reach react-native-web, because React Native's color normalization otherwise drops them before the browser can render them.

| `border-inline-*` / `border-block-*` longhands + shorthands + composite + mode-spanning forms                                                                                                                                                                                             | Applies logical border widths and colors in React Native's horizontal writing mode. Per-edge border styles are browser-only; native builds warn and ignore them.                                                                                                                                                                                                |
| `calc()` / `min()` / `max()` / `clamp()` with static arms                                                                                                                                                                                                                                 | Resolves static math to a React Native-compatible number or percentage.                                                                                                                                                                                                                                                                                         |
| `caret-color`                                                                                                                                                                                                                                                                             | Android maps the first color to the caret only; rn-web lets the browser handle it; iOS keeps the platform default and warns because RN's iOS API tints the caret and selection together.                                                                                                                                                                        |
| `color-mix(in <space>, colorA [p%], colorB [p%])` with literal operands                                                                                                                                                                                                                   | Mixed in the requested space (`lab`, `lch`, `oklab`, `oklch`, `srgb`) per CSS Color Level 5 and rendered as sRGB hex before it reaches React Native / react-native-web. Perceptual spaces use proper round-trip + shorter-arc hue. Dynamic values resolve at render time.                                                                                       |
| CSS system colors (`AccentColor`, `AccentColorText`, `ActiveText`, `ButtonBorder`, `ButtonFace`, `ButtonText`, `Canvas`, `CanvasText`, `Field`, `FieldText`, `GrayText`, `Highlight`, `HighlightText`, `LinkText`, `Mark`, `MarkText`, `SelectedItem`, `SelectedItemText`, `VisitedText`) | Resolve through React Native platform colors where possible, so native UI theme colors adapt to appearance and user settings. rn-web lets the browser resolve the keywords. | `field-sizing: content`                                                                                                                                                                                                                                                                   | Auto-grows a `styled.TextInput` to its content height. Author-declared `min-height` / `max-height` still bound the growth. Explicit `multiline={false}` disables the auto-grow behavior and warns in development. On rn-web, the input renders as a `<textarea>` and the browser handles `field-sizing` where supported.                                        |
| `flex` / `flex-flow` shorthands                                                                                                                                                                                                                                                           | Supports the CSS shorthand syntax for React Native flex grow, shrink, basis, direction, and wrapping behavior.                                                                                                                                                                                                                                                  |
| `font-style: oblique <angle>`                                                                                                                                                                                                                                                             | Maps to `italic` on native (angle dropped + warned); rn-web honors the slant axis natively.                                                                                                                                                                                                                                                                     |
| Generic `font-family` keywords (`cursive`, `emoji`, `fangsong`, `fantasy`, `math`, `monospace`, `sans-serif`, `serif`, `system-ui`, `ui-*`)                                                                                                                                               | Uses an appropriate platform font on iOS and Android; comma-separated fallback lists keep the first family and warn in development.                                                                                                                                                                                                                             |
| `hyphens: none \| manual \| auto`                                                                                                                                                                                                                                                         | Android maps to the system hyphenation frequency; rn-web lets the browser handle it; iOS keeps manual soft-hyphen behavior and warns for `auto`.                                                                                                                                                                                                                |
| `interactivity: inert`                                                                                                                                                                                                                                                                    | Blocks pointer interaction, hides the subtree from assistive technology where React Native exposes that control, and disables focus, text selection, and editing on the styled component. rn-web uses the browser's `inert` behavior.                                                                                                                           |
| `linear()` easing                                                                                                                                                                                                                                                                         | Supports piecewise-linear easing for native animations and transitions.                                                                                                                                                                                                                                                                                         |
| `line-clamp: N`                                                                                                                                                                                                                                                                           | Limits text-like components to the requested number of lines and clips overflow.                                                                                                                                                                                                                                                                                |
| `margin-inline` / `margin-block` / `padding-inline` / `padding-block` / `inset-inline` / `inset-block` shorthands                                                                                                                                                                         | Supports the logical shorthand syntax in React Native's horizontal writing mode.                                                                                                                                                                                                                                                                                |
| `oklch()`, `oklab()`, `lch()`, `lab()` with static channels                                                                                                                                                                                                                               | Converted to sRGB hex before reaching React Native / react-native-web.                                                                                                                                                                                                                                                                                          |
| `outline` shorthand                                                                                                                                                                                                                                                                       | Supports CSS outline width, style, and color shorthand syntax.                                                                                                                                                                                                                                                                                                  |
| `perspective` standalone property                                                                                                                                                                                                                                                         | Applies perspective through the component's transform.                                                                                                                                                                                                                                                                                                          |
| `place-content` / `place-items` / `place-self` shorthands                                                                                                                                                                                                                                 | Supports CSS alignment shorthand syntax for React Native's flex alignment surfaces. The justify-items / justify-self half is browser-only and has no effect on iOS or Android.                                                                                                                                                                                  |
| Relative color syntax (`oklch(from <color> ...)`, `oklab(from <color> ...)`, `lch(from <color> ...)`, `lab(from <color> ...)`)                                                                                                                                                            | Rendered as an sRGB color React Native can display when the base color is knowable, including literal colors, modern color functions, and resolved theme values. `currentColor` bases warn and drop on native because React Native cannot resolve them from inherited styles.                                                                                   |
| `text-shadow` shorthand                                                                                                                                                                                                                                                                   | Supports CSS text-shadow shorthand syntax for React Native text shadows.                                                                                                                                                                                                                                                                                        |
| `text-wrap: [ wrap \| nowrap ] \|\| [ auto \| balance \| stable \| pretty ]`                                                                                                                                                                                                              | `nowrap` renders as one clipped line. `balance` and `pretty` use Android text breaking controls; iOS keeps default line breaking and warns. `stable` is preserved but ignored on native with a warning.                                                                                                                                                         |
| `text-wrap-mode: wrap \| nowrap`                                                                                                                                                                                                                                                          | Supports the mode portion of `text-wrap` independently.                                                                                                                                                                                                                                                                                                         |
| `text-wrap-style: auto \| balance \| stable \| pretty`                                                                                                                                                                                                                                    | Supports the style portion of `text-wrap` independently, with the same platform limits as `text-wrap`.                                                                                                                                                                                                                                                          |
| `transform-box`                                                                                                                                                                                                                                                                           | Ignored on iOS / Android with a one-time development warning because React Native transforms use the view center as their reference box. Use `transform-origin` to move the pivot. rn-web keeps `transform-box` for the browser.                                                                                                                                |
| `transition` shorthand / longhands                                                                                                                                                                                                                                                        | Animates supported property changes automatically; reduced-motion settings collapse durations to zero.                                                                                                                                                                                                                                                          |
| `vertical-align: top \| middle \| bottom \| auto`                                                                                                                                                                                                                                         | Android maps through to RN's text vertical alignment; rn-web maps top/middle/bottom to content alignment while preserving other browser values; iOS warns because RN has no text vertical-align API there.                                                                                                                                                      |

### Dynamic polyfills

These values re-resolve each render against the component tree or the device's current state. Components that don't use any of them pay zero cost.

| Feature                                                                       | What to expect                                                                                                                        |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `@container`, `container-type`, and `container-name`                          | Container queries use the measured size of the nearest or named styled container.                                                     |
| `background-blend-mode`                                                       | On iOS / Android, layered backgrounds render with React Native primitives; rn-web lets the browser handle it.                         |
| `cqw` / `cqh` / `cqmin` / `cqmax` / `cqi` / `cqb`                             | Nearest registered container's `onLayout` size.                                                                                       |
| `em` / `lh` / `rlh` length units                                              | Resolve against inherited font size and line height, so typography-based spacing follows the cascade on native.                       |
| `env(safe-area-inset-top \| right \| bottom \| left)`                         | `useSafeAreaInsets()` (SafeAreaProvider context).                                                                                     |
| `light-dark(light, dark)`                                                     | `Appearance.getColorScheme()` - swaps when OS theme changes.                                                                          |
| `rem` length unit                                                             | `1rem` = 16px by default; `createTheme({ rootFontSize: N })` rescales the ramp. Composes inside `calc()` alongside other static arms. |
| `text-align: start \| end \| match-parent`                                    | Resolves to left or right from the current inherited `direction`.                                                                     |
| `var(--path)` via `createTheme(...)` leaves                                   | Resolves through the active `ThemeProvider` values.                                                                                   |
| `vw` / `vh` / `vmin` / `vmax` / `dvh` / `svh` / `lvh` / `dvw` / `svw` / `lvw` | `Dimensions.get('window')` (re-measures on rotation / resize).                                                                        |

### React Native pseudo-states (equivalent to CSS pseudo-classes)

| CSS                      | React Native mapping                                                                             |
| ------------------------ | ------------------------------------------------------------------------------------------------ |
| `&:active` / `&:pressed` | `Pressable` pressed state.                                                                       |
| `&:disabled`             | Component `disabled` prop.                                                                       |
| `&:focus` / `&:focused`  | `onFocus`/`onBlur` on focusable inputs.                                                          |
| `&:hover`                | `Pressable` hover state (desktop / web) or `onHoverIn`/`onHoverOut` (iOS pencil / desktop-like). |

## Not yet supported (compatibility tracker)

These features depend on upstream React Native primitives that do not yet exist. Each row lists what's missing, the workaround we recommend today, and (where known) the upstream request to follow or support. When a row ships in a React Native release, we'll wire it up and move it out of this section.

### Layout & positioning

| Feature                                                                                                                                                                                                | Why unsupported                                                               | Workaround                                                        |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `anchor-name`, `anchor-scope`, `position-anchor`, `position-area`, `position-try`, `position-try-fallbacks`, `position-try-order`, `position-visibility`, `@position-try`, `anchor()`, `anchor-size()` | No CSS-level anchor positioning primitive on RN.                              | Use popover libraries or `Modal` + measured coordinates.          |
| `display: flow-root`                                                                                                                                                                                   | No block-formatting-context concept on RN.                                    | Use a plain `View`.                                               |
| CSS Grid layout: `display: grid`, `grid-template-*`, `grid-auto-*`, `grid-column`, `grid-row`, `subgrid`, masonry axes                                                                                 | React Native 0.85 exposes no grid layout style props.                         | Use flexbox, nested `View`s, or `FlatList` with multiple columns. |
| `position: fixed`                                                                                                                                                                                      | React Native has no top-layer / viewport-anchored positioning primitive.      | Use `Modal` or a portal-style overlay component.                  |
| `position: sticky`                                                                                                                                                                                     | Only `FlatList`'s `stickyHeaderIndices` exists; no CSS-level sticky.          | Use `FlatList` sticky headers.                                    |
| `reading-flow` / `reading-order`                                                                                                                                                                       | RN's accessibility model uses `accessibilityRole` / focus traversal, not CSS. | Use `accessibilityElementsHidden` / `importantForAccessibility`.  |

### Colors & rendering

| Feature                                                                                       | Why unsupported                                                                  | Workaround                                                                                                                    |
| --------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `@media (color-gamut: srgb \| p3 \| rec2020)`                                                 | No color-space capability query on RN.                                           | Assume sRGB; re-evaluate if wide-gamut support lands upstream.                                                                |
| `@media (dynamic-range: standard \| high)` / `@media (video-dynamic-range: standard \| high)` | No HDR capability query on RN.                                                   | Branch on other signals (explicit user toggle, platform API).                                                                 |
| `@media (forced-colors: none \| active)`                                                      | High-contrast mode is Windows-only on web; no RN equivalent.                     | Use RN's `AccessibilityInfo.isHighTextContrastEnabled()` where available.                                                     |
| `background-image: url(...)` without a blend-mode polyfill path                               | React Native's native background-image parser handles gradients, not image URLs. | Render an `Image` / `ImageBackground`, or use `background-blend-mode` when a blended background layer is the intended effect. |
| `backdrop-filter`                                                                             | No native backdrop-blur primitive on RN.                                         | Use `BlurView` from `@react-native-community/blur` or Skia.                                                                   |
| `border-style: double`                                                                        | Only `solid`, `dashed`, `dotted` supported.                                      | Use a nested border + inner padding.                                                                                          |
| `color-contrast()`                                                                            | Needs rendered-color analysis primitive; only shipped in Safari today.           | Tune an `oklch` lightness threshold manually or select colors via a theme helper.                                             |
| `conic-gradient()`                                                                            | Not implemented in React Native's gradient parser.                               | Use `react-native-linear-gradient` / Skia for custom paint.                                                                   |

## Contributing

If you spot a feature missing from this list, or a workaround that's out of date, open a pull request against this file. When upstream React Native ships a primitive that lets us move a row out of the unsupported section, the same PR should update the polyfill layer so the capability becomes available in styled-components immediately.
